# Snapshot Edit Form: Document-style UX 개선

## 변경 요약

스냅샷 편집 폼을 **대시보드/폼 스타일**에서 **문서 편집 스타일**로 개선했습니다.

## Before vs After

### Before (카드 기반 레이아웃)
- 모든 섹션이 카드(`bg-white rounded-xl border shadow-sm`)로 감싸져 있음
- 카드 안에 또 카드가 중첩 (box-in-box)
- 시각적 노이즈가 많음 (그림자, 테두리, 배경색)
- 폼을 작성하는 느낌

### After (문서 스타일 레이아웃)
- 카드 컨테이너 제거
- 섹션 제목(h2)과 서브섹션 제목(h3)으로 계층 구조 표현
- 타이포그래피와 간격으로 시각적 계층 표현
- 문서를 작성하는 느낌

## 구체적인 변경 사항

### 1. Meta Information
- ❌ 카드 컨테이너 제거
- ✅ 섹션 제목 + 구분선으로 문서 헤더 스타일
- ✅ 라벨 크기 축소 (조용한 컨텍스트)
- ✅ 컴팩트한 그리드 간격

### 2. Past Week / This Week
- ❌ 카드 컨테이너 제거
- ✅ 챕터 제목(h2) 스타일: `text-sm font-bold uppercase tracking-wide`
- ✅ 색상으로 구분: Past Week(파란색), This Week(에메랄드)
- ✅ 섹션 간 충분한 간격 (`mb-10`)

### 3. Tasks / Risks / Collaborators
- ❌ 배경색 박스 제거 (`bg-slate-50/50`)
- ✅ 서브섹션 제목(h3) 스타일로 변경
- ✅ 아이콘 유지
- ✅ 최소한의 컨테이너 (입력 영역만)

### 4. Focus 처리 (읽기 집중도)
- ✅ 포커스된 섹션: `opacity-100` + 좌측 파란색 액센트 라인
- ✅ 비포커스 섹션: `opacity-60` (상호작용은 유지)
- ❌ 강한 테두리, 배경색 블록, 그림자 사용하지 않음

### 5. 간격 및 밀도
- ✅ 같은 서브섹션 내 입력 간 패딩 감소
- ✅ 섹션 간 간격 증가 (`mb-10`)
- ✅ 위에서 아래로 자연스러운 읽기 흐름

## 변경되지 않은 것 (의도적)

- ✅ 모든 기존 단축키 및 헬퍼 UI
- ✅ 작업 추가 / 빠른 추가 / 키보드 UX
- ✅ 진행률 슬라이더 동작
- ✅ 리스크 처리 로직
- ✅ 협업자 로직
- ✅ 데이터 스키마 및 API 호출
