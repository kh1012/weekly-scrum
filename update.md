당신은 Weekly Scrum의 스냅샷 기능을 개선하는 프론트엔드 엔지니어입니다.  
현재 이미 “주간 스냅샷 JSON 데이터 + 카드 리스트 UI”가 존재한다고 가정하고 작업하세요.

## 🎯 목표

1. 전체 스냅샷을 한 번에 볼 수 있는 뷰와, 사람별(Person별)로 필터링해서 볼 수 있는 뷰를 제공한다.
2. “범위(기간)”를 지정해서 여러 주차 스냅샷을 타임라인처럼 왔다 갔다 하며 볼 수 있는 UI를 만든다.
3. 여러 개의 스냅샷을 비교 대상으로 선택해서, 한 화면에서 동시에 비교할 수 있는 비교 뷰를 만든다.
4. 같은 `domain / project / module / feature` 조합을 기준으로 여러 주차의 “전–중–후” 흐름을 비교할 수 있는 UI를 만든다.
5. 스냅샷 작성 시점이 “매주 월요일 오전”이라는 전제를 활용해서:
   - 이전 스냅샷의 `thisWeek` ↔ 현재(중앙) 스냅샷의 `pastWeek`
   - 현재(중앙) 스냅샷의 `thisWeek` ↔ 다음 스냅샷의 `pastWeek`
     를 비교해 **연속성이 유지되고 있는지 / 끊기고 있는지**를 보여주는 “연속성(continuity) 뷰”를 추가한다.
6. 각 스냅샷 카드 우측 상단에 메뉴를 추가하여:
   - (1) 해당 스냅샷 내용을 JSON 형식으로 복사하기
   - (2) 기존 제출 양식인 `submitted-scrum.txt` 포맷으로 변환해서 복사하기  
     (메뉴 라벨: `"Plane Text로 복사하기"` 그대로 사용)

---

## 🔢 데이터/도메인 전제

- 각 스냅샷은 최소한 아래 필드를 가진다고 가정한다. 실제 필드명은 프로젝트에 맞게 매핑해도 된다.
  - `id`: 스냅샷 ID
  - `date`: 스냅샷 기준 날짜 (월요일 날짜)
  - `person`: 담당자 이름
  - `domain`: 도메인 (예: Frontend, Planning, BE 등)
  - `project`
  - `module`
  - `feature`
  - `pastWeek`: 지난 주 내용(Plan/Progress/Risk 등 포함)
  - `thisWeek`: 이번 주 계획/업무 내용
- 이미 “주 단위로 스냅샷들을 로드할 수 있는 API 또는 JSON 파일 구조”가 있다고 가정하고, 필요하다면 헬퍼 함수를 추가해서 가공할 수 있다.

## 🧭 기능 요구사항

### 1. 뷰 모드: 전체 vs 사람별

1. 상단에 뷰 모드 토글/탭 추가
   - 예: `전체 보기` / `사람별 보기`
2. 전체 보기:
   - 선택된 기간(range)에 포함되는 모든 스냅샷을 리스트 or 그리드로 표시
3. 사람별 보기:
   - 사람(person) 필터를 제공하고,
   - 선택된 사람의 스냅샷만 모아서 기간 내에서 보여줌

---

### 2. 범위(기간) 선택 및 주차 이동 UI

1. 기간 선택 컴포넌트

   - 시작 주(또는 날짜), 끝 주(또는 날짜)를 선택할 수 있는 범위 선택 UI
   - 월요일 기준 주차를 인식해서 내부적으로는 “week index”나 ISO 주차 등으로 관리

2. 기간에 포함된 주차 타임라인 표시

   - 예: 상단에 `2025-01-06`, `2025-01-13`, `2025-01-20` 와 같은 주차 단위 버튼/칩을 나열
   - 사용자가 좌우로 스크롤하거나 버튼으로 이전/다음 주차 이동 가능

3. 특정 주차 선택 시
   - 해당 주차를 “중앙(기준) 스냅샷 주차”로 설정
   - 비교/연속성 뷰에서 이 중앙 주차를 기준으로 이전/다음 주차를 함께 계산

---

### 3. 비교 대상 선택 및 멀티 스냅샷 비교 뷰

1. 각 스냅샷 카드에 “비교 대상 선택” 체크박스 또는 토글 추가
   - 사용자가 여러 개의 스냅샷 카드를 체크할 수 있음
2. 하단 또는 상단에 “비교 보기 열기” 버튼 추가
   - 선택된 스냅샷이 2개 이상일 때 활성화
3. 비교 보기 UI
   - 선택된 스냅샷들을 가로로 나란히 배치 (예: 카드 여러 개를 Column layout으로)
   - 공통 필드(domain, project, module, feature, person 등)를 상단에 정렬해서 비교하기 쉽도록 표시
   - 각 카드 내부에 `pastWeek`, `thisWeek` 내용을 같은 구조로 보여줘서 시각적으로 비교 가능하게 설계

---

### 4. Domain/Project/Module/Feature 기준 전·중·후 비교

1. 공통 키 정의
   - 하나의 업무 흐름을 구분하는 키: `(domain, project, module, feature)` 조합
2. 중앙 주차(기준 스냅샷)를 선택했을 때:
   - 같은 키를 가진 이전 주차 스냅샷을 “전(before)”
   - 같은 키를 가진 중앙 주차 스냅샷을 “중(middle)”
   - 같은 키를 가진 다음 주차 스냅샷을 “후(after)”로 간주
3. UI 설계
   - 하나의 업무 흐름을 행(row) 단위로 표현
   - 열(column) 예시:
     - `이전 주차 (pastWeek / thisWeek)`
     - `중앙 주차 (pastWeek / thisWeek)`
     - `다음 주차 (pastWeek / thisWeek)`
   - 각 칸에 해당 스냅샷의 핵심 필드(Plan/Progress/Next 등)를 정리해서 보여준다.
4. 데이터가 비어 있는 경우
   - 어떤 주차에 해당 키 조합이 없다면 “데이터 없음” 또는 회색 처리

---

### 5. thisWeek ↔ pastWeek 연속성 분석 뷰

1. 전제

   - 스냅샷은 “매주 월요일 오전”에 작성된다.
   - 따라서:
     - `이전 스냅샷.thisWeek` 와 `현재 스냅샷.pastWeek`
     - `현재 스냅샷.thisWeek` 와 `다음 스냅샷.pastWeek`
       를 비교하면 업무가 계획대로 이어지는지 확인할 수 있다.

2. 연속성 판단 로직 (기본 버전)

   - 키 기준: `(person, domain, project, module, feature)` 조합이 같으면 같은 업무 흐름으로 본다.
   - 텍스트(Plan/Progress/Next 등) 비교는 우선 단순 매칭 또는 부분 포함 정도로 시작하고,
   - 더 정교한 비교(유사도, 키워드 매칭 등)는 나중에 확장할 수 있도록 구조만 열어둔다.

3. UI 요구사항
   - “연속성 보기” 전용 탭 또는 섹션 추가
   - 각 업무 흐름별로:
     - 이전.thisWeek → 현재.pastWeek 연결 상태
     - 현재.thisWeek → 다음.pastWeek 연결 상태
   - 상태 표시 예시:
     - 초록: 계획 → 진행이 자연스럽게 이어짐
     - 노랑: 부분 일치 / 일부만 이어짐
     - 빨강: 연속성 끊김 (이번 주 pastWeek가 이전 thisWeek와 무관해 보임)
   - 최소한 텍스트 기반 간단 규칙이라도 구현해서,
     - “연속성 있음 / 애매함 / 없음” 3단계로 보여준다.

---

### 6. 스냅샷 카드 메뉴: JSON / Plane Text로 복사하기

1. 각 스냅샷 카드 우측 상단에 옵션 메뉴(… 버튼 등)를 추가한다.
2. 메뉴 항목:
   - `JSON으로 복사하기`
     - 해당 스냅샷 객체를 JSON 문자열로 직렬화하여 클립보드에 복사
   - `Plane Text로 복사하기`
     - 기존 제출 양식인 `submitted-scrum.txt` 포맷으로 변환한 문자열을 생성해서 클립보드에 복사
3. `submitted-scrum.txt` 포맷 예시 (실제 프로젝트에서 사용하는 형식에 맞춰 구현):
   - 예:
     ```
     [Domain / Project / Module / Feature]
     * Name: ...
     * Past Week
       * ...
     * This Week
       * ...
     * Risk: ...
     * RiskLevel: ...
     ```
   - 실제 사용 중인 텍스트 포맷 규칙이 있다면, 그 규칙에 맞춰 포맷터 함수를 구현한다.

## 📐 기술적 요구사항

1. 기존 스냅샷 리스트/카드 컴포넌트를 재사용하고, 가능한 한 새로운 페이지/컴포넌트는 “뷰 모드, 비교 모드, 연속성 모드” 단위로 분리해서 설계한다.
2. 상태 관리:
   - 선택된 기간
   - 뷰 모드(전체/사람별/연속성 등)
   - 선택된 사람(person)
   - 비교 대상 스냅샷 목록 (ID 배열)
   - 중앙(기준) 주차
3. URL 또는 LocalStorage 연동은 기존 정책을 따른다.
   - 가능하다면 `?range=...&view=...&person=...` 형태로 쿼리스트링에 반영해서 공유 가능한 상태로 만든다.

## ✅ 완료 기준 (Acceptance Criteria)

- 기간을 지정하면 해당 범위의 주차들이 타임라인/리스트로 표시된다.
- “전체/사람별” 모드를 전환해도 기간/선택 정보가 유지된다.
- 여러 스냅샷을 선택해서 비교 보기로 들어가면, 한 화면에서 동시에 내용을 비교할 수 있다.
- 같은 `(domain, project, module, feature)` 조합에 대해, 전–중–후 흐름을 일관된 레이아웃으로 확인할 수 있다.
- 연속성 뷰에서:
  - 이전.thisWeek ↔ 현재.pastWeek
  - 현재.thisWeek ↔ 다음.pastWeek
    가 시각적으로 표현되며, 최소한 3단계(연속/애매/단절) 정도의 상태가 구분된다.
- 각 스냅샷 카드에서:
  - JSON 복사가 정상 동작한다.
  - `Plane Text로 복사하기` 메뉴를 누르면 `submitted-scrum.txt` 포맷의 문자열이 클립보드에 복사된다.
- 기존 스냅샷 페이지의 성능/렌더링에 치명적인 회귀(regression)가 없다.
