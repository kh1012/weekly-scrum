## 📌 3. 리스크 및 후속 조치 계획

### Risk 1: 외부 API 응답 지연 (L2) - 조해용

- **발생 원인**: M-Connector 동기화 API에서 외부 API 의존성 존재
- **전파 범위**: 
  - FE/Spreadsheet 모듈 (김서연)
  - FE/Import 모듈 (김정빈)
  - 전체 동기화 기능 안정성
- **후속 조치**: 
  - 외부 API timeout 정책 명문화
  - fallback 로직 또는 retry with exponential backoff 구현 검토

### Risk 2: 알림 메시지 유실 가능성 (L2) - 김건우

- **발생 원인**: 큐 소비 속도 개선 중 일부 메시지 유형에서 유실 징후
- **전파 범위**: 
  - MOTIIV 알림 시스템 전체
  - 한내경(waiting-on) 온보딩 테스트 일정
- **후속 조치**:
  - 메시지 유실 케이스 로깅 강화
  - retry 정책 확장 (dead letter queue 도입 검토)

### Risk 3: CORS 정책 충돌 (L2) - 하성열

- **발생 원인**: 플러그인 sandbox 구조에서 cross-origin 통신 이슈
- **전파 범위**: Idea-forge 플러그인 아키텍처 전체
- **후속 조치**:
  - 메시징 시스템 통합 전 CORS 정책 사전 정의
  - postMessage 기반 통신 규격 문서화

### Risk 4: 엣지 케이스 재현 어려움 (L2) - 김정빈

- **발생 원인**: Import 오류 케이스가 특정 조건에서만 발생
- **전파 범위**: M-Desk Import 기능 안정성
- **후속 조치**:
  - 재현 환경 snapshot 저장
  - 자동화된 edge case 테스트 시나리오 작성

### Risk 5: 브라우저별 렌더링 편차 (L1) - 김서연

- **발생 원인**: 셀 렌더링 최적화 후 브라우저 호환성 이슈
- **전파 범위**: MOTIIV Spreadsheet 크로스 브라우저 지원
- **후속 조치**: 주요 브라우저(Chrome, Safari, Firefox, Edge) 렌더링 테스트 수행

### Risk 6: 템플릿-팀프로젝트 정책 충돌 (L1) - 윤재웅

- **발생 원인**: 템플릿 정책과 기존 팀프로젝트 모듈 정책 간 불일치 가능성
- **전파 범위**: MOTIIV Workspace 전체 IA
- **후속 조치**: IA 전체 정합성 검증 시 팀프로젝트 담당자(최은진) 리뷰 포함
