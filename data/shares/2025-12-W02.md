## 📌 2. 즉시 결정해야 할 항목 리스트업

### 1. 셀 병합 기능의 edge-case 처리 방향 결정

- **현재 상태**: Progress 40%, 단일 셀 병합 시 데이터 손실 리스크(L2) 존재
- **지연 시 비용/영향**:
  - Spreadsheet 모듈 전체 안정성에 직접 영향
  - 사용자 데이터 손실 시 신뢰도 하락
- **의사결정 옵션**:
  - (A) edge-case별 데이터 백업 로직 선 적용 후 병합 기능 배포
  - (B) 데이터 손실 가능 케이스 UI 경고 표시 후 사용자 선택에 위임

### 2. 권한 검증 로직과 도메인 모델 정합성 확보

- **현재 상태**: Progress 50%, 도메인 모델 충돌 리스크(L2), 윤재웅(waiting-on)
- **지연 시 비용/영향**:
  - 권한 부여 케이스에서 보안 취약점 발생 가능
  - BE 권한 로직과 기획 정책 불일치 시 전체 재작업 필요
- **의사결정 옵션**:
  - (A) 윤재웅 기획 정책 확정 우선 → 권한 로직 후속 적용
  - (B) 현재 도메인 모델 기준으로 권한 로직 선 구현 → 정책 확정 후 조정

### 3. 조해용 작업 우선순위 결정 (데이터 정합성 vs 대용량 업로드)

- **현재 상태**: 두 작업 모두 Progress 50~65%, 각각 L2 리스크 존재
- **지연 시 비용/영향**:
  - 데이터 정합성: 김정빈 FE 작업(상태 UI) 지연
  - 대용량 업로드: 김정빈 FE 연동 지연
- **의사결정 옵션**:
  - (A) 데이터 정합성 우선 (상태 UI가 먼저 필요한 경우)
  - (B) 대용량 업로드 우선 (Import 기능 출시 임박 시)

## 📌 3. 리스크 및 후속 조치 계획

### Risk 1: 단일 셀 병합 시 데이터 손실 (L2) - 김서연

- **발생 원인**: edge-case 처리 중 특정 병합 패턴에서 원본 데이터 유실 가능
- **전파 범위**: MOTIIV Spreadsheet 전체, 사용자 데이터 신뢰성
- **후속 조치**:
  - 병합 전 셀 데이터 임시 저장 로직 추가
  - 병합 취소(Undo) 기능 동시 구현 검토

### Risk 2: DB 락 지연 발생 (L2) - 조해용

- **발생 원인**: 데이터 정합성 점검 시 동시 쓰기 작업과 충돌
- **전파 범위**: M-Desk Data 모듈, 전체 쓰기 작업 성능
- **후속 조치**:
  - 락 타임아웃 정책 명문화
  - 점검 작업의 off-peak 시간대 스케줄링 검토

### Risk 3: 메모리 spike 발생 (L2) - 조해용

- **발생 원인**: 대용량 업로드 시 특정 파일 구조에서 메모리 급증
- **전파 범위**: M-Desk Import 기능, 서버 안정성
- **후속 조치**:
  - streaming 업로드 청크 사이즈 최적화
  - 메모리 임계값 초과 시 자동 중단 로직 추가

### Risk 4: 도메인 모델 충돌 (L2) - 김건우

- **발생 원인**: 권한 검증 로직이 기존 도메인 모델 구조와 불일치
- **전파 범위**: MOTIIV User 모듈, 전체 권한 체계
- **후속 조치**:
  - 기획(윤재웅) 정책 확정 후 도메인 모델 재검토
  - 권한 로직 변경 시 영향 범위 사전 분석

### Risk 5: shared memory 충돌 (L2) - 하성열

- **발생 원인**: worker-thread 간 shared memory 접근 경합
- **전파 범위**: Idea-forge Plugin 아키텍처
- **후속 조치**:
  - SharedArrayBuffer 사용 패턴 검토
  - 뮤텍스/세마포어 기반 동기화 로직 적용

### Risk 6: 반자동화 도구 안정성 (L2) - 한내경

- **발생 원인**: QA 라벨링 도구의 예외 케이스 처리 미흡
- **전파 범위**: Team 프로세스, QA 효율성
- **후속 조치**:
  - 도구 안정성 테스트 케이스 추가
  - fallback 수동 라벨링 프로세스 문서화

### Risk 7: 필드 overflow (L2) - 김희성

- **발생 원인**: Profile UI에서 긴 텍스트 필드 처리 미흡
- **전파 범위**: MOTIIV Profile 사용자 경험
- **후속 조치**:
  - 텍스트 truncation 정책 수립
  - 툴팁 또는 확장 표시 UI 적용
