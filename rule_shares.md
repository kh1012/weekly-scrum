입력 데이터는 ./data/scrum을 기반으로 한다.
위 디렉토리 기준으로 가장 최근에 생성된 json 데이터를 기준으로 아래 지침대로 내용을 정리해서
./data/shares 디렉토리 하위에 json 파일과 같은 이름으로 지정된 .md 파일을 생성해줘.

---

당신의 역할:
단순 요약이 아니라, "팀 운영 의사결정에 필요한 패턴·리스크·로드 변화"를 감지하는 분석관이다.

아래 6가지 항목을 반드시 출력하라.
해석 없이 나열하거나 요약하는 것을 금지한다.  
각 항목은 ‘왜 그렇게 판단했는지’ 기반 근거를 포함하라.

---

### 1. 이번 주 각 분야별 핵심 진행 요약 (기획 / 개발 / 디자인 / 콘텐츠)

- 요약이 아니라 "의미 있는 변화만" 보고한다.
- 반복적 업무·보고용 문구는 제외한다.
- 팀 간 의존성, 병목이 감지되면 표시한다.

### 2. 즉시 결정해야 할 항목 리스트업

- "미룰수록 비용이 커지는 항목" 관점으로 뽑는다.
- 선택지 또는 의사결정 포인트를 명확히 제시한다.

### 3. 리스크 및 후속 조치 계획

- 리스크 레벨(L1~L3)을 포함한다.
- 리스크 발생 원인 / 전파 범위 / 후속 조치를 각각 출력한다.

### 4. 리소스 편중 및 재분배 시그널

- 특정 팀원·도메인에 로드가 몰린 패턴 감지
- 결손 리소스 또는 과투입 리소스를 명시
- 다음 스프린트에서 조정해야 하는 포인트 제안

### 5. 다음 주 우선순위 Top 3

- “중요도×긴급도” 기준으로 점수화하여 선정한다.
- 왜 Top 3인지 이유 포함.

Strict Mode:

- 논리적 모순이 있는 답변을 금지한다.
- 데이터에 없는 내용을 꾸며내지 말고, 대신 “데이터 부족”이라고 명시하라.
- 판단 근거를 숨기지 말고 반드시 서술하라.

이제 위 기준에 따라 결과를 생성하라.
